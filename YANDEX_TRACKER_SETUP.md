# Настройка Яндекс Трекера

## Шаг 1: Создайте организацию в Яндекс Трекере

1. Перейдите на https://tracker.yandex.ru/
2. Войдите в свой аккаунт Яндекса
3. Если у вас нет организации, создайте новую

## Шаг 2: Получите OAuth токен

1. Перейдите на https://tracker.yandex.ru/settings/tokens
2. Нажмите "Получить токен"
3. Разрешите доступ к Трекеру
4. Скопируйте полученный токен

## Шаг 3: Узнайте ID организации

1. Перейдите на https://tracker.yandex.ru/admin/orgs
2. Скопируйте ID вашей организации (числовое значение)

## Шаг 4: Создайте очередь VOICEBOT

1. Перейдите в https://tracker.yandex.ru/
2. Нажмите "Создать очередь"
3. Введите ключ очереди: `VOICEBOT`
4. Заполните название: "Voice Calendar Bot"
5. Выберите шаблон: "Разработка"
6. Нажмите "Создать"

## Шаг 5: Обновите .env файл

Добавьте полученные данные в файл `.env`:

```bash
YANDEX_TRACKER_TOKEN=ваш_oauth_токен
YANDEX_TRACKER_ORG_ID=ваш_id_организации
YANDEX_TRACKER_QUEUE=VOICEBOT
```

## Шаг 6: Создайте задачи

Запустите скрипт для создания всех задач проекта:

```bash
python scripts/create_tracker_tasks.py
```

## Альтернатива: Работа без Яндекс Трекера

Если вы не хотите использовать Яндекс Трекер, вы можете:

1. Закомментировать вызовы Tracker API в коде
2. Использовать локальное отслеживание задач (TODO.md)
3. Пропустить автоматизацию с Test Agent

Проект будет работать и без интеграции с Трекером, но вы потеряете:
- Автоматическое создание задач
- Связывание коммитов с задачами
- Автоматическую валидацию тестов через Test Agent
